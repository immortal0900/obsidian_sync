# Obsidian Vault 양방향 동기화 데몬 프로젝트

## 목표
로컬 Obsidian 볼트와 Google Drive 간 양방향 자동 동기화 데몬을 Python으로 구현한다. Obsidian이나 VSCode 없이 OS 레벨 백그라운드 프로세스로 상시 동작해야 한다.

## 핵심 요구사항

### 동기화 메커니즘
- **로컬→Drive**: `watchdog` 라이브러리로 파일 변경을 이벤트 기반 감지 → 변경된 파일만 Google Drive API로 업로드
- **Drive→로컬**: Google Drive Changes API를 주기적 폴링(1분 간격) → 변경분만 다운로드
- 충돌 해결: last-write-wins (마지막 수정 시간 기준)

### 프로젝트 구조
볼트 밖에 별도 프로젝트로 구성. 코드는 Git, 설정은 기기별 분리.

```
obsidian-sync/
├── core/
│   ├── watcher.py          # watchdog 파일 감지
│   └── drive_sync.py       # Google Drive 동기화
├── hooks/
│   ├── __init__.py          # 훅 레지스트리
│   └── sync_hook.py         # 동기화 훅 (1단계)
├── main.py                  # 데몬 진입점
├── config.example.yaml      # 설정 템플릿 (Git 포함)
├── config.yaml              # 실제 설정 (Git 제외)
└── .gitignore
```

### 훅 시스템 설계 (확장성 핵심)
watcher가 파일 변경 감지 시 등록된 훅을 순서대로 실행하는 구조.
현재는 `sync_hook`만 구현하지만, 나중에 아래와 같이 확장할 수 있어야 한다:
- `blog_hook`: 특정 폴더 변경 시 마크다운→HTML 변환 후 배포 (2단계)
- `llm_hook`: LLM 기반 자동 태깅, 볼트 정리 (3단계, LangGraph 연동)

```yaml
# config.yaml 예시
watch_paths:
  - path: ~/ObsidianVault
    hooks: [sync]
  # 나중에 추가:
  # - path: ~/ObsidianVault/blog
  #   hooks: [sync, blog_convert]
  # - path: ~/ObsidianVault/notes
  #   hooks: [sync, llm_tagging]
```

### 데몬 배포
- Windows: `pythonw.exe` 또는 NSSM으로 서비스 등록
- Linux/Mac: systemd 서비스 등록
- 부팅 시 자동 실행, 콘솔 창 없이 백그라운드 동작

### 다른 기기 적용
1. `git clone`으로 코드 받기
2. `config.example.yaml` → `config.yaml` 복사 후 로컬 볼트 경로 수정
3. Google Drive 인증 토큰은 기기별 생성 (Git 미포함)
4. 데몬 등록

## 제약사항
- `config.yaml`, Google Drive 토큰 파일은 `.gitignore`에 포함
- 순환 트리거 방지: 동기화 작업 자체가 watchdog 이벤트를 발생시키지 않도록 처리
- 디바운싱: 파일 변경 감지 시 즉시 동기화하지 않고, 마지막 변경 후 N초(기본 5초)간 추가 변경이 없을 때 동기화 실행 (Obsidian 자동저장으로 인한 API 호출 폭주 방지)
- 현재 1단계(동기화만)에 집중. 훅 인터페이스만 확장 가능하게 설계하고, LLM 관련 구현은 하지 않는다.

## 기술 스택
- Python 3.12
- watchdog (파일 시스템 감지)
- Google Drive API (google-api-python-client)
- PyYAML (설정 파일)
- 패키지 관리: uv
